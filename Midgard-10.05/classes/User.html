static get(mgd, n_params, parameters)¶
Parameters:
  • mgd (Midgard.Connection) – Midgard.Connection instance
  • n_params (int) – number of parameters
  • parameters (GObject.Parameter) – GObject.Parameter with Midgard.User properties
Returns:

new Midgard.User instance or None in case of failure
Return type:

Midgard.User

Fetch Midgard.User object from storage. At least ‘login’ and ‘authtype’ property are required to be set in parameters.

Cases to return None:

  • ‘login’ or ‘authtype’ properties do not exist in given parameters (MGD_ERR_INVALID_PROPERTY_VALUE)
  • There’s no user object which match given parameters (MGD_ERR_NOT_EXISTS)
  • More than one record found in database (MGD_ERR_INTERNAL)

New in version 9.09.

log_in()¶
Returns:True if user has been logged in, False otherwise
Return type:bool

Logs in user instance, if given one is valid. A valid user object must have (at least) guid set. Which means, Midgard.Object must be fetched from database. Either with Midgard.User.get() or with Midgard.User.query().

This method silently returns with success when given user is already logged in.

Cases to return False:

  • User’s guid is not set ( MGD_ERR_INVALID_PROPERTY_VALUE )

New in version 9.09.

log_out()¶

Return type:bool

Cases to return False:

  • There’s no user logged in (MGD_ERR_INTERNAL)
  • User is not recently logged in (MGD_ERR_INTERNAL)

New in version 9.09.

set_person(person)¶
Parameters:person (Midgard.Object) – Midgard.Object instance
Returns:True on success, False otherwise
Return type:bool

Associates given Midgard.Object person with self Midgard.User. Sets person property and updates user storage record.

Midgard.User self takes ownership of the given Midgard.Object (‘midgard_person’ type) reference, and increases person’s object reference count.

See Midgard.User.update() for returned error details.

update()¶
Returns:True on success, False otherwise
Return type:bool

Updates user storage record

Cases to return False:

  • User with such login and authentication type already exists ( MGD_ERR_DUPLICATE )
  • User’s guid is not set ( MGD_ERR_INVALID_PROPERTY_VALUE )
  • ‘authtype’ property is empty or None ( MGD_ERR_INVALID_PROPERTY_VALUE )
  • User record hasn’t been found ( MGD_ERR_INTERNAL )
  • Failed to update storage record ( MGD_ERR_INTERNAL )
  • ‘authtype’ property value is invalid ( MGD_ERR_INTERNAL )

New in version 9.09.
