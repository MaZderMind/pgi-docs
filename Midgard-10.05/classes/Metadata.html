class Midgard.Metadata(**kwargs)¶
Bases:Midgard.DBObject
Abstract:No
Structure:Midgard.MetadataClass
static new(object)¶
Parameters:object (Midgard.DBObject) – Midgard.Object for which metadata is created
Returns:newly allocated midgard_metadata instance
Return type:Midgard.Metadata

Creates new midgard_metadata instance for the given Midgard.Object instance.

Do not use GObject.Object.new() as metadata constructor. Midgard.Object pointer is internally assigned as a pointer to midgard object for which particular metadata object instance was created.

Midgard.Metadata object has two “kinds” of properties. The first one is settable ( and overwritten ) only by metadata implementation. The second one is freely settable by application. In this case midgard core keep value of such property “as is”.

Do not free Midgard.Metadata object’s memory as it is automatically freed when particular Midgard.Object object’s instance memory is freed.
