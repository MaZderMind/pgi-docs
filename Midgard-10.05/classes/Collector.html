static new(mgd, typename, domain, value)¶
Parameters:
  • mgd (Midgard.Connection) – Midgard.Connection instance
  • typename (str) – name of given class, which collector is initialized for
  • domain (str) – collection’ domain
  • value (GObject.Value) – domain’s constraint value
Returns:

Midgard.Collector instance, or None on failure
Return type:

Midgard.Collector

typename should be any Midgard.DBObject derived class name. domain is property name which is registered for given class, and should not be unique per object or record. In other words, domain is a common property (and value) for group of objects expected in collection. value must be of domain property type, and is owned by Midgard.Collector instance. If you need reuse given value, make a copy.

Cases to return None:

  • mgd Midgard.Connection instance is invalid
  • typename is not registered in GObject.GType system or it’s not Midgard.DBObject derived
  • domain property is not registered for given typename class
  • value is of invalid type

merge(mc, overwrite)¶
Parameters:
  • mc (Midgard.Collector) – Midgard.Collector instance
  • overwrite (bool) – whether overwrite collector’s keys
Returns:

True if collections has been merged, False otherwise
Return type:

bool

If third overwrite parameter is set as True then all keys which exists in self and mc collector’s instance will be oberwritten in self colection instance. If set as False , only those keys will be added, which do not exist in self collection and exist in mc collection.

Cases to return False:

  • Second argument is not valid Midgard.Collector
  • mc, Midgard.Collector has no keys collection

set(key, subkey, value)¶
Parameters:
  • key (str) – key name for which subkey&@value pair should be set
  • subkey (str) – property name which is a subkey
  • value (GObject.Value) – value for given subkey
Returns:

True, if key’s value has been set, False otherwise
Return type:

bool

Cases to return False:

  • self, Midgard.Collector instance is invalid
  • subkey property name is not registered for collestor’s class
  • value is of invalid type

If the key is already added to Midgard.Collector then its value (as subkey&amp;value pair) is destroyed and new one is set. In other case new key and its subkey&amp;value pair is added to collector.

Key used in this function is a value returned ( or set ) for collector’s key. Keys are collection of values returned from property fields. Subkey is an explicit property name.

GObject.Value value argument is owned by Midgard.Collector. If value should be reused, its copy should be passed as method argument.

set_key_property(key, value)¶
Parameters:
  • key (str) – property name which must be a key
  • value (GObject.Value) – optional value of key’s constraint
Returns:

True if key has been set, False otherwise
Return type:

bool

If value is explicitly set to None , then all key property name’s records are selected from database and set in internal collector’s resultset. If not, key property name and its value is used as constraint to limit selected records from database. In latter case add_value_property method should be invoked. GObject.Value value passed as third argument is owned by Midgard Collector. If value should be reused, its copy should be passed to constructor.

Cases to return False:

  • self Midgard.Collector instance is invalid
  • key property is not registered for the typename class
  • value is of invalid type
  • key property is already set for self Midgard.Collector
