Class Details¶

class GSystem.SubprocessContext(**kwargs)¶
Bases:GObject.Object
Abstract:No

This class contains a set of options for launching child processes, such as where its standard input and output will be directed, the argument list, the environment, and more.

While the GSystem.Subprocess class has high level functions covering popular cases, use of this class allows access to more advanced options. It can also be used to launch multiple subprocesses with a similar configuration.

open_pipe_read()¶
Raises:GLib.Error
Returns:True on success, False on error (and error will be set)
out_stream:A newly referenced output stream
out_fdno:File descriptor number for the subprocess side of the pipe
Return type:(bool, out_stream: Gio.InputStream, out_fdno: int)

This allows you to open a pipe between the parent and child processes, independent of the standard streams. For this function, the pipe is set up so that the parent can read, and the child can write. For the opposite version, see GSystem.SubprocessContext.open_pipe_write().

The returned out_fdno is the file descriptor number that the child will see; you need to communicate this number via a separate channel, such as the argument list. For example, if you’re using this pipe to send a password, provide --password-fd=<fdno string>.

Property Details¶

GSystem.SubprocessContext.props.argv¶
Name:argv
Type:[str]
Default Value:[]
Flags:READABLE, WRITABLE

Array of arguments passed to child process; must have at least one element. The first element has special handling - if it is an not absolute path ( as determined by GLib.path_is_absolute() ), then the system search path will be used. See GLib.SpawnFlags.SEARCH_PATH.

Note that in order to use the Unix-specific argv0 functionality, you must use the setter function gs_subprocess_context_set_args_and_argv0(). For more information about this, see GLib.SpawnFlags.FILE_AND_ARGV_ZERO.

New in version 2.36.
