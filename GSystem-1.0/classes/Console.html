class GSystem.Console(**kwargs)¶
Bases:GObject.Object
Abstract:No

First, this class offers API to access the standard input and output/error, streams as Gio.InputStream and Gio.OutputStream respectively.

In the case where the process is connected to a controlling terminal, the GSystem.Console.get() API is available, which exposes a number of additional features such as no-echo password reading.

classmethod get()¶
Returns:The console instance, or None if not interactive
Return type:GSystem.Console

If the current process has an interactive console, return the singleton GSystem.Console instance. On Unix, this is equivalent to isatty(). For all other cases, such as pipes, sockets, /dev/null, this function will return None.

New in version 2.36.

begin_status_line(line, cancellable)¶
Parameters:
  • line (str) – String to output
  • cancellable (Gio.Cancellable or None) –
Raises:

GLib.Error
Return type:

bool

The primary use case for this function is to output periodic “status” or “progress” information. The first time this function is called, line will be output normally. Subsequent invocations will overwrite the previous.

You must invoke GSystem.Console.end_status_line() to return the console to normal mode. In particular, concurrent use of this function and the stream returned by GSystem.Console.get_stdout() results in undefined behavior.

end_status_line(cancellable)¶
Parameters:cancellable (Gio.Cancellable or None) –
Raises:GLib.Error
Return type:bool

Complete a series of invocations of GSystem.Console.begin_status_line(), returning the stream to normal mode. The last printed status line remains on the console; if this is not desired, print an empty string to clear it before invoking this function.
