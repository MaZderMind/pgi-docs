Returns:a newly created Clutter.Texture

Create a new Clutter.Texture with the specified image. Adds the image to the cache if the image had not been previously loaded. Subsequent calls with the same image URI/path will return a new Clutter.Texture with the previously loaded image.

insert(uri, texture)¶
Parameters:
  • uri (str) – A URI or local file path
  • texture (object) – A #CoglHandle to a texture

Inserts a texture into the texture cache. This can be useful if you want to cache a texture from a custom or unhandled URI type, or you want to override a particular texture.

If the image is already in the cache, this texture will replace it. A reference will be taken on the given texture.

New in version 1.2.

insert_meta(uri, ident, texture, destroy_func)¶
Parameters:
  • uri (str) – A URI or local file path
  • ident (object or None) – A unique identifier
  • texture (object) – A #CoglHandle to a texture
  • destroy_func (GLib.DestroyNotify) – An optional destruction function for ident

Inserts a texture that’s associated with a URI into the cache. If the metadata already exists for this URI, it will be replaced.

This is useful if you have a widely used modification of an image, for example, an image with a border composited around it.

New in version 1.2.
